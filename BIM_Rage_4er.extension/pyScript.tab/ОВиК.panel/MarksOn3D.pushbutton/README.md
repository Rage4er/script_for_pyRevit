# 📐 Марки на 3D видах и планах

**Версия:** 2.0  
**Автор:** Rage  
**Назначение:** Автоматическое размещение марок на 3D-видах и планах этажей

---

## 📋 Описание

Скрипт предназначен для автоматической расстановки марок элементов ОВиК (воздуховоды, воздухораспределители, оборудование) на выбранных видах Revit.

### Основные возможности:

- ✅ **Поддержка 3D видов и планов этажей**
- ✅ **Раздельные настройки марок** для 3D и планов
- ✅ **Раздельный выбор категорий** для 3D и планов
- ✅ **Автоматическая фильтрация воздуховодов** (по одному на группу система/сечение)
- ✅ **Гибкие настройки размещения** (ориентация, выноска, смещение)
- ✅ **Сохранение настроек** между запусками
- ✅ **Подробное логирование** процесса выполнения

---

## 🚀 Быстрый старт

### 1. Запуск скрипта

1. Откройте проект Revit
2. Перейдите на вкладку **pyRevit** → **ОВиК** → **Марки на 3D**
3. Откроется главное окно скрипта

### 2. Настройка видов (Вкладка 1)

- Выберите виды из списка (3D виды и/или планы этажей)
- Используйте поиск для быстрого нахождения видов
- Виды отсортированы по имени

### 3. Выбор категорий (Вкладка 2)

- Отметьте категории элементов для маркировки
- Можно выбрать разные категории для 3D и планов
- Доступные категории:
  - Воздуховоды
  - Гибкие воздуховоды
  - Изоляция воздуховодов
  - Воздухораспределители
  - Арматура воздуховодов
  - Оборудование

### 4. Выбор марок (Вкладка 3)

- **Две отдельные таблицы** для 3D видов и планов
- Дважды кликните по строке для выбора семейства и типа марки
- Настройки сохраняются для следующего запуска

### 5. Настройки размещения (Вкладка 4)

| Параметр | Описание |
|----------|----------|
| **Ориентация** | Горизонтальная или вертикальная ориентация марки |
| **Использовать выноску** | Добавить выноску к марке |
| **Случайное смещение** | Случайное направление смещения марки от элемента |

### 6. Выполнение (Вкладка 5)

- Проверьте сводку настроек
- Нажмите **"Выполнить"** для запуска
- Следите за прогрессом выполнения
- После завершения проверьте результат в логе

## ⚙️ Технические детали

### Алгоритм работы:

1. **Сбор элементов** по выбранным видам и категориям
2. **Фильтрация воздуховодов** — оставляется по одному самому длинному воздуховоду на группу (система + сечение)
3. **Проверка существующих марок** — пропускаются элементы с уже размещёнными марками
4. **Размещение марок** с учётом настроек смещения и ориентации
5. **Смена типа марки** на выбранный пользователем
6. **Сохранение настроек** в JSON-файл

### Формат файла настроек (`tag_defaults.json`):

```json
{
  "Воздуховоды": {
    "family_3d": "Марка воздуховода 3D",
    "type_3d": "Стандартный",
    "family_plan": "Марка воздуховода План",
    "type_plan": "Компактный"
  },
  "Воздухораспределители": {
    "family_3d": "...",
    "type_3d": "...",
    "family_plan": "...",
    "type_plan": "..."
  }
}
```

## 🐛 Известные ограничения

- **Тип конца выноски** (свободный/присоединённый) нельзя изменить программно — это настройка семейства марки
- Для корректной работы необходимы загруженные в проект семейства марок соответствующих категорий

## 📝 Примеры использования

### Сценарий 1: Маркировка только на 3D виде

1. На вкладке 1 выберите только 3D вид
2. На вкладке 3 настройте марки только для 3D
3. Запустите скрипт

### Сценарий 2: Разные марки для 3D и планов

1. Выберите и 3D вид, и план этажа
2. На вкладке 3 настройте разные типы марок:
   - Для 3D: подробная марка с размерами
   - Для плана: компактная марка

### Сценарий 3: Массовая маркировка нескольких видов

1. Выберите несколько видов (Ctrl+клик)
2. Настройте категории и марки
3. Запустите скрипт — все виды будут обработаны

## 🆘 Устранение неполадок

### Проблема: Марки не создаются

**Решение:**
- Проверьте, что выбранные виды не являются шаблонами
- Убедитесь, что в проекте есть семейства марок выбранных категорий
- Проверьте лог на наличие ошибок

### Проблема: Не сохраняется тип марки

**Решение:**
- Убедитесь, что типоразмер марки активен в проекте
- Проверьте, что категория марки соответствует категории элемента

### Проблема: Ошибка кодировки при сохранении настроек

**Решение:**
- Скрипт использует UTF-8 кодировку через модуль `codecs`
- Убедитесь, что путь к проекту не содержит недопустимых символов
