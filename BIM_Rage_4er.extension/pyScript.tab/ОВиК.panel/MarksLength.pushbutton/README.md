# 📏 MarksLength — Автодлина марок

**Версия:** 2.0  
**Автор:** Rage  
**Назначение:** Автоматическая корректировка длины полки марок на видах в зависимости от содержимого

---

## 📋 Описание

Скрипт автоматически рассчитывает и устанавливает оптимальную длину полки (выноски) марки на основе длины текста значения параметра. Это делает марки более читаемыми и аккуратными, избегая как чрезмерно длинных, так и слишком коротких выносок.

### Основные возможности:

- ✅ **Поддержка 3D видов и планов этажей**
- ✅ **Раздельные настройки параметров** для 3D и планов
- ✅ **Настраиваемый коэффициент расчёта** длины полки
- ✅ **Две отдельные таблицы** выбора параметров
- ✅ **Сортировка видов по имени**
- ✅ **Сохранение настроек** между запусками
- ✅ **Подробное логирование** через pyRevit

## 🚀 Быстрый старт

### 1. Запуск скрипта

1. Откройте проект Revit
2. Перейдите на вкладку **pyRevit** → **ОВиК** → **Автодлина марок**
3. Откроется главное окно скрипта

### 2. Выбор видов (Вкладка 1)

- Выберите виды из списка (3D виды и/или планы этажей)
- Виды отсортированы по имени
- Используйте кнопки "Выбрать все" / "Снять выбор"
- Поиск по названию вида

### 3. Настройка параметров (Вкладка 2)

#### Коэффициент длины полки

```
Длина полки (мм) = количество символов × коэффициент + 1
```

**Значение по умолчанию:** 1.6  
**Диапазон:** 0.5 - 5.0

**Пример:**
- Текст: "В1.1-1" (6 символов)
- Коэффициент: 1.6
- Расчёт: `6 × 1.6 + 1 = 10.6 мм` → **11 мм** (округление)

#### Раздельные таблицы параметров

| 3D виды - параметры | Планы - параметры |
|---------------------|-------------------|
| Выберите параметр для расчёта длины на 3D видах | Выберите параметр для расчёта длины на планах |
| Двойной клик для выбора | Двойной клик для выбора |

**Поддерживаемые категории:**
- Воздуховоды
- Гибкие воздуховоды
- Изоляция воздуховодов
- Воздухораспределители
- Арматура воздуховодов
- Оборудование

### 4. Выполнение (Вкладка 3)

- Нажмите **"Выполнить"**
- Следите за прогрессом в прогресс-баре
- Результаты отобразятся в текстовом поле
- Логирование через pyRevit (если доступно)

## ⚙️ Технические детали

### Алгоритм работы:

1. **Сбор марок** с выбранных видов
2. **Получение значения параметра** (instance или type)
3. **Расчёт требуемой длины полки**: `ceil(len(text) × coefficient + 1)`
4. **Поиск существующего типоразмера** с подходящей длиной
5. **Создание нового типоразмера** если не найден
6. **Применение типоразмера** к марке
7. **Сохранение настроек** в JSON

### Формула расчёта:

```python
required_length = math.ceil(char_count * coefficient + 1)
```

Где:
- `char_count` — количество символов в значении параметра
- `coefficient` — настраиваемый коэффициент (по умолчанию 1.6)
- `+1` — дополнительный запас в мм

### Параметры для расчёта:

Скрипт ищет параметр в следующем порядке:
1. **Instance параметр** элемента
2. **Type параметр** семейства элемента

### Формат файла настроек (`tag_defaults.json`):

```json
{
  "length_coefficient": 1.6,
  "Воздуховоды": {
    "param_3d": "Имя системы",
    "param_plan": "Диаметр"
  },
  "Воздухораспределители": {
    "param_3d": "Комментарии",
    "param_plan": "Имя типа"
  }
}
```

### Логика именования типоразмеров:

Скрипт использует **последнее подчёркивание с цифрами** для разделения префикса и длины:

| Исходное имя | Разбиение | Префикс | Новое имя (13мм) |
|--------------|-----------|---------|------------------|
| `Размер / Расход_11` | `["Размер / Расход", "11"]` | `Размер / Расход` | `Размер / Расход_13` ✅ |
| `Размер_13` | `["Размер", "13"]` | `Размер` | `Размер_13` ✅ |
| `Марка_8` | `["Марка", "8"]` | `Марка` | `Марка_8` ✅ |

## 🐛 Известные ограничения

- **Работает только с марками**, имеющими параметр "Длина полки" / "Shelf Length"
- **Создаёт новые типоразмеры** в семействах марок

## 🆘 Устранение неполадок

### Проблема: Марки не изменяются

**Причина:** У марки нет параметра "Длина полки"

**Решение:**
- Используйте семейства марок с параметром "Shelf Length" или "Длина полки"
- Проверьте название параметра в семействе марки

### Проблема: Создаётся слишком много типоразмеров

**Причина:** Большой разброс длин текста

**Решение:**
- Уменьшите коэффициент (например, до 1.2)
- Округляйте значения параметров

### Проблема: Параметры не сохраняются

**Причина:** Нет доступа на запись в папку скрипта

**Решение:**
- Проверьте права доступа к папке
- Файл `tag_defaults.json` создаётся в папке скрипта
